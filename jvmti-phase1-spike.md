# axon-jvmti Phase 1 — spike report

> Phase 1 deliverable for Operation-NSP JVM bridge research.

## 1. Executive summary

Phase 1 native JVMTI bridge is **working on JDK 17 (Eclipse Temurin)** on Windows. The attach loader + native agent enumerate `com.nelieo.probe.*` classes with fields, methods, constructors, instance counts, and primitive `instance_sample` values.

**Phase 2 recommendation:** Wire `probe_jvm()` in `axon-jvmti` mirroring `axon-v8`, add persistent IPC channel and SSF normalizer. Keep the **native JVMTI agent** as the default path (lower attach overhead, richer heap introspection) with the Java Instrumentation agent as a fallback for restricted environments.

## 3. In-process vs IPC

JVMTI must stay **in-process** inside the target JVM. IPC is host↔agent transport only.

| Transport | Mechanism | Phase 1 status |
|-----------|-----------|----------------|
| stdout NDJSON | Agent writes to JVM stdout | Primary path — verified |
| Named pipe | `\\.\pipe\axon-jvmti-{pid}` | Windows spike — agent client + host server in `axon-jvmti` |

Host `--ipc` spawns a pipe server before attach; agent duplicates each NDJSON line to the pipe when `ipc=1` is set. Latency is comparable to stdout for the test app (single burst). Run `scripts/jvmti-ipc-bench.ps1` for p50 comparison.

## 7. Risks and mitigations

| Risk | Mitigation |
|------|------------|
| `jvmti-bindings` null slice panics | `jvmti_safe.rs` guards zero-count arrays |
| Windows `loadAgentLibrary` path | Strip `.dll` in `agent_path.rs` |
| JDK 21 dynamic attach | Document `-XX:+EnableDynamicAgentLoading` |
| Agent stderr noise on attach | Expected — NDJSON on target JVM stdout |
| 100x re-attach memory | `scripts/jvmti-leak-test.ps1` — 100 cycles, 0 failures on JDK 17 |
| Enterprise Spring STW under load | **Mitigated** — `scripts/jvmti-enterprise-stress.ps1` + Tier A/B fixtures; see `docs/jvmti-enterprise-stress-results.md` |

## 8. Deallocate audit

All JVMTI string allocations go through `jvmti-bindings` helpers that call `Deallocate`. Additional safe wrappers in `jvmti_safe.rs` guard null pointer arrays for `GetClassFields`, `GetClassMethods`, and `GetObjectsWithTags`.

## 10. Phase 2 architecture recommendation

1. **`axon-watcher` probe slot** — implement `probe_jvm()` using `axon-jvmti::attach_agent`, mirroring V8 lifecycle.
2. **Persistent IPC** — keep stdout for debugging; production uses named pipe or shared memory ring buffer.
3. **`axon-types` integration** — map `ClassDescriptor` → SSF substrate nodes; reuse `ActionKind::JvmMethodInvocation` for Phase 2 JNI calls.
4. **Obfuscation** — ingest ProGuard mapping files before LLM labeling.
5. **Java agent fallback** — ship `fixtures/java-agent-spike` pattern as optional JAR for environments blocking native agents.
