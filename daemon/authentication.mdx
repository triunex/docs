## Overview

By default, the NSP daemon requires every API request to include a valid API key in the `X-Substrate-Key` header. This prevents unauthorized local processes or scripts from accessing your daemon.

The daemon authenticates requests via cloud verification with [platform.nelieo.com](https://platform.nelieo.com). Verified keys are cached in memory for sub-millisecond authentication performance.

<Warning>
  The daemon will log a warning on startup if `auth.require_key = false`. **Never** disable auth in production environments.
</Warning>

---

## Getting an API Key

All NSP API keys are generated from the Nelieo platform dashboard:

1. Go to [platform.nelieo.com](https://platform.nelieo.com)
2. Sign in to your Nelieo account
3. Generate your API key

Your key format:
```text
nsp_live_sk_a1b2c3d4e5f6789...
```

---

## Authenticating Requests

Pass your API key in the `X-Substrate-Key` HTTP header for REST calls, or pass it to the SDK.

### cURL

```bash
curl -H "X-Substrate-Key: nsp_live_sk_a1b2c3d4e5f6..." \
     http://localhost:7842/substrate/v1/apps
```

### Python SDK

```python
import nelieo_nsp as nsp

# Pass key explicitly
async with nsp.NSPClient(api_key="nsp_live_sk_a1b2c3d4...") as client:
    apps = await client.list_apps()

# Or via environment variable NSP_API_KEY=nsp_live_sk_a1b2c3d4...
async with nsp.NSPClient() as client:
    apps = await client.list_apps()
```

## How Key Verification Works

```
Agent Request (X-Substrate-Key: nsp_live_sk_...)
       │
       ▼
Local NSP Daemon (localhost:7842)
       │
       ├── Check in-memory verification cache (sub-ms hit)
       │
       └── If cache miss → POST https://platform.nelieo.com/api/verify
               │
               ├── 200 OK → Cache key & permit request
               └── 401 Unauthorized → Return 401 JSON error
```

---

## Authentication Errors

| Status | Error Code | Description |
|--------|-----------|-------------|
| `401` | `missing_key` | No `X-Substrate-Key` header present |
| `401` | `invalid_key` | Key invalid or revoked on platform.nelieo.com |
| `401` | `disabled_key` | Key disabled on platform console |

## Disabling Auth (Development Only)

```toml
# DEVELOPMENT ONLY
[auth]
require_key = false
```

```bash
# Or via environment variable
AXON_AUTH__REQUIRE_KEY=false axon-daemon.exe run
```

When auth is disabled, the daemon prints a continuous warning log:

```text
⚠️  NSP DAEMON RUNNING WITHOUT AUTHENTICATION — ALL REQUESTS ACCEPTED ANONYMOUSLY
```
